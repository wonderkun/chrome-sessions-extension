# 账号多开助手

基于 **Chrome Manifest V3** 的侧栏扩展：在本地采集、分组管理网页 **Cookie 登录态**，一键写回浏览器并打开对应页面；支持 **同站点多账号**（按标签绑定会话）、可选 **后台保活** 刷新登录。数据保存在本机 `chrome.storage.local`，不向第三方服务器上传会话内容。

**版本**：见 [`public/manifest.json`](public/manifest.json) 中的 `version` 字段。

## 它能做什么

| 能力 | 说明 |
|------|------|
| **采集会话** | 保存当前标签的标题、图标、URL、Cookie 快照（通过 `chrome.cookies`，非页面脚本篡改 `document.cookie`）。 |
| **分组管理** | 自定义分组、▲▼ 调整顺序、删除分组（含组内会话，原地二次确认）；会话可拖入分组或详情内下拉切换；阿里云 **按产品自动分组**。 |
| **切换账号** | 点击会话：先写入快照 Cookie，再 **打开 / 复用 / 新建** 标签进入对应站点。 |
| **标签绑定** | 采集时记录 `tabId`；**同 URL 多账号** 时，自动写 Cookie 只作用于「采集时用的那个标签」，互不串号。 |
| **自动写 Cookie** | 标签 **切换到前台** 时，若该标签已绑定会话且当前页与会话 **同源**，自动写入该会话 Cookie（同标签内继续点击链接 **不会** 重复注入）。 |
| **清除站登录态** | 一键删除当前页相关 Cookie 并刷新，便于同站重新登录；已保存会话可用「打开」恢复。 |
| **保活** | 定时或手动：写 Cookie → 带 Cookie 请求保活 URL → 回读浏览器 Cookie 更新快照。 |
| **阿里云增强** | 控制台页 **浮标** 快速打开侧栏；采集时尽力解析 **主账号** 显示名（DOM 变更可能导致失效）。 |

侧栏与浮标 UI 会跟随系统 **`prefers-color-scheme`** 在浅色 / 深色间切换。

## 工作原理（简）

1. **写入 Cookie**：使用扩展 API **`chrome.cookies.set`** 写入浏览器全局 Cookie 存储；页面下一次请求会自然携带。
2. **会话 ↔ 标签**：每条会话可有可选字段 **`boundTabId`**（采集或「打开」成功后写入）。关闭对应标签时自动清除绑定。
3. **自动注入条件**：`tabs.onActivated` + 已开启「自动应用 Cookie」+ 当前 URL 为 `http(s)` + `boundTabId` 匹配 + 与会话 `url` **同源**（协议 + 主机 + 端口）。

更完整的需求与边界见 [docs/cookie-session-tab-binding.md](docs/cookie-session-tab-binding.md)。

## 安装

### 从源码构建

```bash
npm install
npm run build:extension
```

`build:extension` 会生成图标、执行 `tsc` + `vite build`，并运行 `scripts/prepare-dist.mjs` 产出可加载的 **`dist`** 目录。

### 加载到 Chrome

1. 打开 `chrome://extensions`
2. 开启右上角 **开发者模式**
3. **加载已解压的扩展程序** → 选择本仓库的 **`dist`** 目录

## 快速上手

1. 在任意 **http(s)** 页面点击工具栏扩展图标，打开 **侧栏**。
2. 登录目标站点后，点击 **采集当前页**，生成一条会话（并与当前标签 **绑定**）。
3. 需要切换账号时：先采集另一个账号（建议在 **另一个标签** 中登录后采集），再点击对应会话 **打开**，或依赖 **自动写 Cookie**（切换回已绑定的标签时）。
4. 用 **分组** 整理会话；**双击标题** 可改名（含阿里云 SLS / OSS 等场景的默认标题规则）。

## 功能说明

### 采集与会话

- Cookie 来自 **`chrome.cookies.getAll({ url })`**，反映浏览器当前 Cookie 仓，**不是**扩展里上一份快照的回放。
- **同一标签再次采集**：更新 **已绑定该标签** 的那条会话（不新建第二条），避免同一 `tabId` 对应多条会话。
- 若无法读取标签 `id`（极少见），会提示错误，不进行采集。

### 清除站登录态（免点网站注销）

侧栏 **「清除站登录态」** 会按当前标签 URL 调用与采集相同的 **`getAll({ url })` 范围**，逐个 **`chrome.cookies.remove`**，然后 **刷新当前标签**。适合在新标签里换账号登录、又不想点站点「退出」（避免服务端把其它端会话一并作废）的场景。

**重要**：Cookie 在 **同一 Chrome 用户配置下全局共享**，清除后 **同一站点的所有标签** 都会失去该站在浏览器里的登录态；侧栏里 **已保存的会话快照仍在**，可对原账号会话再点 **「打开」** 把 Cookie 写回并恢复页面。

### 标签绑定与自动写 Cookie

- 仅当会话带有 **`boundTabId`** 且你从其他标签 **切回** 该标签、且页面与会话 **同源** 时，才会自动写入该会话 Cookie。
- **不会**在每次 `loading` / 每次 URL 变化时注入，避免同标签内操作被反复覆盖。
- 侧栏可关闭「自动应用 Cookie」；旧数据无 `boundTabId` 的会话不会参与上述自动逻辑，仍可用 **打开** 或 **保活**。

### 手动「打开」页面

顺序为：**优先绑定标签** → **按规范化 URL（去 `#`）查找已有标签** → **新建标签**。

- 绑定标签仍在且与会话 URL（规范化后）一致：聚焦并 **刷新** 或 **导航** 到保存的 URL。
- 绑定标签仍在但用户已把该页 **导航到别处**：**不**改写该标签，**新开** 标签打开会话，并把绑定迁到新标签。
- 成功打开后，会 **去除其他会话上重复的同一 `boundTabId`**，保证一个标签只对应一条绑定会话（数据一致）。

### 分组与标题

- **未分组** / 自定义分组；删除命名分组时会 **一并删除组内所有会话**（垃圾桶二次确认）。
- **分组标题行**：左侧 **▲ / ▼** 调整分组顺序（写入 `groups_v1`）；右侧垃圾桶先点一次，在原地出现 **取消 / 确定删除**：会 **删掉该分组及组内全部会话**（不再移入未分组）。
- **会话卡片** 左侧 **⋮⋮** 可拖到 **未分组** 区域或 **命名分组**（标题栏或展开区）上松手即可换组；详情里下拉框仍可改分组。
- **双击** 会话标题进入编辑；阿里云 SLS、OSS 等有专用默认标题推断逻辑（见代码 `lib/aliyun.ts`）。

### 刷登录保活

对每条会话（可单独关闭「定时刷登录」）：

1. 将快照 Cookie 写回浏览器  
2. 对 **保活 URL** 发起 `GET`（默认同会话 URL 去 hash，可在卡片中自定义）  
3. 使用 `fetch` + `credentials: include` + `no-cors` 尽量带上站点 Cookie  
4. 再用 `getAll` 将 **最新 Cookie** 写回快照  

**定时**：约每 **3 小时** 顺序执行全部未关闭保活的会话（会话之间约 **2.5 秒** 间隔，减轻同域抢写 Cookie）。侧栏可 **立即刷登录** / **全部刷登录**。

### 阿里云：自动分组与浮标

- 在 `*.console.aliyun.com` 采集时，按子域映射到分组（如百炼、ECS、SLS、OSS）；未映射子域使用 `阿里云 · <子域大写>`。
- 在 `*.aliyun.com` / `aliyun.com` 页面注入 **右下角浮标**，点击可在 **当前窗口** 打开侧栏。
- **主账号** 显示名：优先解析控制台账号信息区 DOM，失败则回退到页面文本正则；控制台改版可能导致读不到。

## 权限

| 权限 | 用途 |
|------|------|
| `storage` | 会话、分组、扩展偏好持久化 |
| `cookies` | 读取 / 写入站点 Cookie |
| `tabs` | 查询标签、打开 / 刷新 / 绑定 `tabId` |
| `sidePanel` | 侧栏界面 |
| `activeTab` | 配合当前标签采集 |
| `alarms` | 定时保活 |
| `host_permissions: <all_urls>` | 对任意站点读写 Cookie 与发起保活请求（请仅在信任环境使用） |

## 限制与隐私

- **同一 Chrome 用户配置只有一套 Cookie 罐**。多条会话若域名重叠，**保活按顺序执行**时，后执行的会话会覆盖浏览器中的 Cookie，与「多账号同时在线」的浏览器模型一致；切换账号依赖你主动 **打开 / 切换标签** 或采集时机。
- 若站点强依赖 **前端逻辑**、**CSP** 或 **风控**，仅靠扩展写 Cookie + `fetch` 可能无法续期，需自行调整保活 URL 或改用页面内操作。
- `no-cors` 下无法读取响应体；保活以「请求已发出且携带 Cookie」为一次尝试。
- 会话数据仅存 **本机**；请妥善保管设备与 Chrome 配置。

